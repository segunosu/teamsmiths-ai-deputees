# PRODUCT_SPEC.md — AI Alpha OS

## What it is
**AI Alpha OS** is an AI-powered operating-partner platform that helps **one senior consultant /
partner / director** identify, deliver, govern, measure, and monetise AI-driven value creation across
multiple companies — operating with the leverage of a 10–20 person team.

It is **not** a generic CRM, **not** a generic project-management app, **not** a governance-only tool.
It is a **consultant leverage engine** that produces decisions, not documents.

## Primary user
A senior consultant/partner/director who must, alone, run origination, diagnosis, delivery,
governance, measurement, monetisation, and learning across a portfolio of clients.

## Modules
0. **Client Search & Acceptance Engine** — *existing Stage 0.* Find → score → prioritise →
   snapshot/outreach/proposal → accept/nurture/reject → onboard. (`/agile-ai-alpha`)
1. **Accepted Prospect → Client Conversion** — turn an Accepted/Onboarded company into a client +
   default engagement + diagnostic/opportunity/KPI shells, preserving all prior data.
2. **Client Workspace** — single view of a client: engagements, diagnostics, opportunities, sprints,
   governance, evidence, KPIs, value ledger, monetisation, reports, patterns, activity, narrative.
3. **Engagement Workspace** — a unit of paid/standing work (Diagnostic, 90-Day Sprint, Operating
   Partner Subscription, 4Ps Review, Agile AI Review, Build Sprint, Continuous Improvement).
4. **AI Alpha Diagnostic** — inputs → summary, findings, top opportunities/risks, readiness, value
   potential, recommended 90-day focus; links to 4Ps + Agile AI maturity; human review.
5. **AI Opportunity Engine** — discover/score/prioritise opportunities (transparent priority score),
   select for sprint, generate story backlog, link to KPI/risk/4Ps.
6. **Agentic Scrum Delivery** — sprints + stories on a board with acceptance criteria, definition of
   done, golden tests, owner type (human/AI roles), AI vs human points, and enforced review gates.
7. **4Ps Governance & Evidence** — risks → controls → evidence → framework mappings, all linked to the
   four Ps; readiness mapping only, never certification.
8. **KPI Attribution & Value Ledger** — baseline/target/actual, uplift, financial value, attribution
   confidence, client-agreed state; distinguishes estimated / validated / client-agreed / disputed.
9. **Monetisation** — fixed fee / subscription / fee+upside / gain-share / warrant-equity; triggers and
   invoice status; human review before any commercial claim.
10. **Portfolio Learning** — reusable, anonymised-by-default patterns and benchmarks across engagements.
11. **Reports** — Markdown reports (diagnostic, 90-day plan, sprint review, monthly value, 4Ps,
    KPI attribution, board pack, portfolio summary, monetisation) with human review before sharing.
12. **Command Centre** — one cockpit answering: who needs attention, what to convert, which diagnostics
    are incomplete, top opportunities, blocked sprints, stories needing review, KPI movement,
    monetisable value, top risks, missing evidence, reports to review, patterns captured, and the
    current consultant leverage factor.

## Narrative Intelligence Layer (cross-cutting)
At every key point the consultant can write freeform observations (`aaos_narrative_notes`). An
"Analyse Observation" action proposes structured items (signal, risk, opportunity, KPI hypothesis,
evidence gap, score adjustment, story, task, etc.) into a **review panel** — Accept / Edit / Reject.
Nothing is auto-applied. The original narrative is always preserved and traceable to anything derived
from it.

## Success definition
One senior consultant can move a company from **accepted prospect → client → engagement → diagnostic →
opportunity → governed sprint delivery → KPI movement → monetisation → portfolio learning → reports**,
all from one Command Centre, with full traceability and visible human-review gates.
